## 1. 一句话定义

> **用户输入目的地（"黄石 5 天"），10 秒内生成一张图文并茂、可交互、可分享的单文件 HTML 旅行活地图——让不爱看文字攻略的人也能秒懂行程。**

---

## 2. 背景与机会

### 2.1 现状痛点
| 现有产品 | 问题 |
|---|---|
| 小红书攻略 | 散乱、无地理结构、无法整体规划 |
| 马蜂窝/穷游 | 字多、PDF 死板、不交互 |
| Google My Maps | UI 工程师风、无审美、无叙事节奏 |
| OTA 行程单 | 表格化、无视觉重点、无"必去"vs"备选"分层 |
| ChatGPT 文字行程 | 纯文本墙、无地图、无视觉差异 |

### 2.2 用户真实痛点（来自原型验证 + 朋友反馈）
- **"陪我去的家人不看字"**——爸妈/小孩/对象只想看图就懂今天去哪
- **"攻略改一下太麻烦"**——临时少 1 天想重排，文字攻略全部失效
- **"分享给同伴像甩论文"**——发个 5000 字攻略对方根本不看
- **"博主想做产品但不会画图"**——KOL 攻略文章变现率低

### 2.3 我们的差异化洞察
传统攻略产品都在解"信息量"问题，但用户真正缺的是 **"一眼就懂的视觉化叙事"**——
> **地图 + emoji + Day 路线 + 一句话理由** 比 5000 字 markdown 更易传播、更易记忆、更易决策。

### 2.4 市场规模（估算）
- 中国自由行人数（2025）：~6 亿人次
- 国际自由行人数（2025）：~3 亿人次
- 假设 5% 渗透率 × 3 元/张活地图（订阅或单次付费）= **~135 亿规模**
- 早期不收费，靠 KOL / OTA B 端付费 + 微信传播

## 3. 目标用户

### 3.1 用户画像

| 角色 | 占比目标 | 核心场景 | 付费意愿 |
|---|---|---|---|
| 🎯 **P0 自由行规划者** | 60% | 输入"想去 XX N 天"，把生成的地图发给同伴 | 中（可付费去广告/高级模板）|
| 🎯 **P1 旅行 KOL/博主** | 25% | 把攻略文章一键转活地图，作为文末"可视化总结"嵌入或分享 | 高（B 端订阅 $99/月）|
| 🎯 **P2 OTA / 旅行社** | 15% | 给客户的行程单从 PDF 升级为活地图链接 | 极高（按客户量计费）|

### 3.2 反用户画像（明确不服务）
- 专业徒步/登山者（需要等高线、轨迹文件，我们做不到）
- 商务出行（无观光元素）
- 没有目的地概念的"散步式"游客

## 6. 功能需求（按优先级）

### F1. 输入层 · P0
- **F1.1** 文本输入：目的地 + 天数 + 偏好标签
- **F1.2** 偏好枚举：家庭/深度/刺激/慢游/美食/亲子/蜜月（多选）
- **F1.3** 可选：出发地（用于推荐航线 + 倒推抵达日玩什么）
- **F1.4**（v2）URL 粘贴：粘贴小红书/马蜂窝攻略链接，自动提取
- **F1.5**（v2）图片上传：截图攻略，OCR + LLM 提取

### F2. POI 智能生成 · P0
- **F2.1** LLM（Claude）按目的地+天数+偏好生成 POI 列表
  - 每天 3-5 个 POI，总数 15-30
  - 每个 POI 含：name、en、coords（lng/lat）、category、day、num、time、desc（150 字内）、tags（必看/独家/亲子 等）、icon（emoji）、color
- **F2.2** Geocoding：调 Mapbox / Google Geocoding 校正经纬度
- **F2.3** Category 自动分类（22 类预设，emoji + 色票映射）
- **F2.4** Day 路线优化：按经纬度做 TSP 简化，避免每天来回 100km
- **F2.5**（v2）实时校验：POI 是否关闭、季节性是否合理（如黄石 11 月主路封）

### F3. 地图渲染 · P0
- **F3.1** **底图策略**（核心决策点 ⚠️）
  - **方案 A · 模板库**：预先手工绘制 100 个热门目的地 SVG（黄石/大岛/京都/巴黎...）→ 美但有限
  - **方案 B · 自动生成**：用 Mapbox Static 或 OpenStreetMap 自动出底图 → 全覆盖但缺温度
  - **推荐方案 C · 混合**：
    - Tier 1（30 个超热门）：手绘 SVG 模板
    - Tier 2（100 个常见）：Mapbox 抽象风格化（去掉路名/POI，只留轮廓）
    - Tier 3（其他）：Mapbox 默认风格
- **F3.2** POI 标记：彩色 pin + emoji + 编号 + 名称
- **F3.3** Day 路线：彩色虚线 + 流动动画
- **F3.4** 图例：彩色 dot + emoji + 类别名
- **F3.5** Day Tabs：切换天数高亮路线
- **F3.6** 移动端响应式：1050px 以下单列布局
- **F3.7**（v2）拖拽编辑 POI 位置

### F4. 交互细节 · P0
- **F4.1** 点 POI → 右侧详情面板（emoji + 中文名 + 英文名 + tag chips + 150 字理由 + 时段建议）
- **F4.2** Day Tab 切换 → 非当天 POI 淡化 + 当天路线高亮
- **F4.3**（v1.5）"下一站"按钮 → 唤起 Google Maps / 高德导航
- **F4.4**（v2）语音朗读（TTS）→ 点喇叭听 POI 介绍
- **F4.5**（v2）30 秒短视频预览 → 抓 YouTube/抖音 嵌入
- **F4.6**（v2）儿童模式 → 卡通图标 + 简化语音

### F5. 行程编辑 · P1
- **F5.1** 拖拽天数顺序：把 Day 3 整体调到 Day 1
- **F5.2** 增/删 POI：从"备选 POI 池"拖入，或右键删除
- **F5.3** 改文案：内联编辑 POI 名/描述
- **F5.4** 改天数：5 天改成 4 天，自动合并/删除最不重要的 POI
- **F5.5** 版本历史：撤销/重做

### F6. 分享与导出 · P0
- **F6.1** 短链：`livemap.app/x7k2`（云端存储）
- **F6.2** 下载 HTML：单文件 ≤ 100KB，离线可用
- **F6.3** 微信分享卡片：自动生成 5:4 缩略图
- **F6.4**（v2）小红书长图：导出 9:16 长图（含全部 POI 列表）
- **F6.5**（v2）微信小程序卡片
- **F6.6**（v2）PDF 导出（给爸妈打印）

### F7. 个性化与登录 · P1
- **F7.1** 不登录可生成（短链 7 天有效）
- **F7.2** 微信/Google 登录可保存历史（永久）
- **F7.3** 收藏地图、协作编辑（与同伴共享编辑权限）

### F8. 后台与运营 · P2
- **F8.1** 模板库后台（运营手工补充 Tier 1 手绘 SVG）
- **F8.2** POI 修正反馈（用户报错 → 人工/AI 修）
- **F8.3** 热门目的地 dashboard（哪些目的地被生成最多）

## 8. 技术架构

### 8.1 系统架构图（文字）
```
[用户浏览器]
     ↓ HTTP
[Next.js 前端 (Vercel)]
     ↓ API
[API Gateway (Node.js)]
     ↓
  ┌──┴──┬──────┬──────────┬────────┐
  ↓     ↓      ↓          ↓        ↓
[Claude] [Mapbox] [OpenAI  [Supabase] [Cloudflare
 提POI    地理    Embedding 用户/地图   R2 静态
 LLM     编码    去重      存储        资源]
```

### 8.2 关键技术选型

| 模块 | 技术 | 理由 |
|---|---|---|
| 前端 | Next.js 15 + Tailwind | SSR 利于 SEO、组件化 |
| LLM | Claude 4.5 Sonnet | 中文/景点知识强、JSON 输出稳 |
| 地理编码 | Mapbox（10 万次/月免费）| 比 Google 便宜 5 倍 |
| 底图 | Mapbox Static + 自研 SVG 模板库 | 见 F3.1 |
| 数据库 | Supabase（PostgreSQL）| 自带 Auth + 短链 |
| 静态托管 | Cloudflare R2 + Pages | HTML 单文件存储分享链接 |
| 部署 | Vercel | 国内访问需 + 香港 CDN 节点 |

### 8.3 核心 Prompt 设计（LLM 提 POI）
```python
PROMPT = """
你是资深旅行规划师。为以下需求生成详细 POI JSON：
- 目的地：{destination}
- 天数：{days}
- 偏好：{preferences}
- 出发地：{origin}

输出严格 JSON 格式，含 15-30 个 POI：
[
  {
    "id": "string",
    "day": 1-N,
    "num": 1-N,
    "name": "中文名",
    "en": "English name",
    "lat": 44.428,
    "lng": -110.588,
    "category": "geyser|waterfall|wildlife|...",
    "icon": "🌋",
    "color": "#d4854e",
    "desc": "150字内：为什么去 + 看什么 + 实用 tip",
    "tags": ["必看", "亲子"],
    "time": "上午|中午|下午|傍晚|晚上"
  }
]

规则：
1. 每天 3-5 个 POI，按地理就近排序避免折返
2. 每天必含 1 个"必看" tag
3. desc 要有"独家信息"（如门票/最佳时间/避坑），不只是百科介绍
4. 经纬度精确到 0.001 度
"""
```

## 9. MVP 分阶段范围

### v0.1 · "手工模板验证"（1 周）⭐ 当前阶段
- ✅ 大岛 7 天（已有）
- ✅ 黄石 5 天（已有）
- ➕ 京都 6 天 / 冰岛 7 天 / 北海道 5 天（手工补 3 个）
- **目的**：发给 20 个朋友看，统计"愿意分享"率 → 验证 PMF 信号

### v0.2 · "AI 生成 + 固定模板"（2 周）
- 落地页 + 输入框
- Claude API 调用，POI JSON 输出
- 渲染到现有 SVG 模板（限 5 个热门目的地）
- 不支持编辑，只读+分享
- **目的**：跑通"输入→生成→分享"全链路

### v0.3 · "Mapbox 底图通用化"（3 周）
- 接 Mapbox 自动出抽象底图
- 支持 50 个目的地
- 短链 + 微信卡片分享
- **目的**：能覆盖任意目的地

### v1.0 · "可编辑 + 商业化"（6 周）
- 拖拽编辑 POI / Day
- 用户登录 + 历史保存
- KOL/OTA B 端订阅
- **目的**：开始挣钱

### v1.5 · "无文字体验"（4 周）
- 语音 TTS / 视频预览 / 一键导航
- 儿童模式
- **目的**：覆盖"不爱看字"长尾人群

## 11. 风险与开放问题

### 11.1 关键风险

| 风险 | 等级 | 缓解 |
|---|---|---|
| LLM 生成 POI 错误（坐标/营业状态）| 🔴 高 | 后台人工抽检 + Mapbox 校正 + 用户报错入口 |
| 自动底图美感不如手绘 | 🟡 中 | Tier 1 热门目的地手绘兜底 |
| 微信无法直接打开 HTML 链接 | 🟡 中 | 转跳 H5 中转页 + 小程序版本 |
| 抄袭/复刻容易 | 🟡 中 | 靠"模板库 + KOL/OTA B 端关系"建壁垒 |
| 商业化变现路径不清晰 | 🔴 高 | 早期免费扩规模，B 端先于 C 端 |

### 11.2 待决策问题（标 🔴 需 PM/老板拍板）

1. 🔴 **底图策略最终选哪个**？手绘 vs Mapbox vs 混合（影响开发量 3-10x 差异）
2. 🔴 **首批热门目的地选哪 5 个**？建议：黄石、京都、冰岛、北海道、巴厘岛
3. 🟡 **是否支持英文**？v0 中文为主，国际化推迟
4. 🟡 **小程序 vs Web 优先级**？建议 Web 先（验证更快），小程序 v1.5 跟进
5. 🟡 **是否做编辑功能**？编辑会大幅增加开发量，可考虑 v1.0 才上
